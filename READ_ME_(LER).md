# Curso de Levantamento de Requisitos 60hrs (80 aulas)

## Módulo 1 - Introdução ao Curso

### Módulo 2 - Levantamento de Requisitos (continuação)

- O que é Levantamento de Requisitos
    - Transforma necessidade do cliente/usuário em informações técnicas

- Requisitos Funcionais e Não Funcionais
    - RF - O que o sistema vai fazer
    - RNF - Como o sistema vai fazer

- Requisitos, Regras e Restrições
    - Requisitos - O que o dev vai fazer e a parte técnica do sistema
    - Regras - As necessidades dos clientes
    - Restrições - Limitações impostas pela linguagem/lei

- Modelo de Documentação de Requisitos

|      Campo      |         Conteúdo         |             Exemplo             |
|-----------------|--------------------------|---------------------------------|
|        ID       |          Código          |              RF-001             |
|      Título     |           Valor          |           Tela e Login          |
|    Descrição    |   O Ator deve ser Capaz  | O Cliente conseguirá fazer Login|
|    Prioridade   |    Valor da Prioridade   |              Alta               |
|Regras Vinculadas|Outras Regras Relacionadas|     RN-001 - Banco de Dados     |

### Técnicas de levantamentos de requisitos
  - Briefing
  - BrainStorm
  - Questionario
  - Entrevista
  - Etnografia
  - Workshop
  - Prototipagem

## Briefing
- Coleta inicial de dados
    - Não existe um formato pré definido
    - Permite a coleta de diversas informações do cliente
         - Manual de identidade visual (Cores, logo, slogan) 
    - Concorrentes
- Prazos 
- Orçamentos
- Identidade da Empresa
- Público Alvo

## Brainstorming
  - Reunião com o objetivo de solucionar problemas
  - Podem ser estruturados ou não estruturados
  - Reunião curta com foco bem definido
  - Ferramentas
      - MIRO
      - Mentimeter
      - Onenote

## Questionario
- Perguntas direcionadas para o escopo do projeto (Documentação final do projeto)
- Formas de fazer o questionario
    - Reunião com perguntas diretas (Qualitativos)
    - Formularios de coleta de dados (Quantitativos)

## Entrevista
- Instrumento ultilizado para Coleta de dados com o Usuário do projeto
- Entrevistas podem ser mais formais e Infomrais
- Nessa etapa é necessario ter um MVP (Produto Mínimo Viável)

## Etnografia 
- Observação da Cultura da empresa passivamente (Como é atendido e o relacionamento interno e externo)

## Workshop
- Ocorrem na apresentação de uma MVP, para um grupo seleto de pessoas (Stakeholders) com intuito de absorver a reação do público ao produto/projeto
    - Evento, Feiras, workshops de produtos especificos

## Prototipagem
- Criação de um elemento de design para simulação de um projeto, com o intuito de testar funiconalidadese elementos graficos antes de começar a escrever códigos
    - Os prototipos podem ser de:
        - Baixa fidelidade (Rafe) - Rascunho de um desenho que pode ser feito a mão
        - Média fidelidade (Rough) - Desenhos no computador para marcação de espaços e layouts, não prescisa de texto nem cores
        - Alta fidelidade - Protótipo focado nas funcionalidades iguais aos do projeto que será desenvolvido posteriormente em código
